## ⚡ 프로젝트 개요

**MiroFish**는 다중 에이전트 기술 기반의 차세대 AI 예측 엔진입니다. 현실 세계의 시드 정보(예: 속보, 정책 초안, 금융 신호)를 추출하여 고해상도의 평행 디지털 세계를 자동으로 구축합니다. 이 공간 내에서 독립적인 인격, 장기 기억 및 행동 논리를 갖춘 수천 개의 지능형 에이전트가 자유롭게 상호 작용하고 사회적 진화를 겪습니다. "전지적 시점"에서 변수를 동적으로 주입하여 미래의 궤적을 정확하게 추론할 수 있습니다 — **디지털 샌드박스에서 미래를 리허설하고 수많은 시뮬레이션 후 최종 결정을 내리세요**.

> 당신이 해야 할 일: 시드 자료(데이터 분석 보고서 또는 흥미로운 소설 이야기)를 업로드하고 자연어로 예측 요구 사항을 설명하세요.</br>
> MiroFish가 반환하는 것: 상세한 예측 보고서와 심층적으로 상호 작용이 가능한 고해상도 디지털 세계

### 우리의 비전

MiroFish는 현실을 매핑하는 군집 지능 미러를 구축하는 데 전념합니다. 개별 상호 작용에 의해 촉발되는 집단적 창발을 포착하여 기존 예측의 한계를 돌파합니다.

- **거시적 관점**: 우리는 의사 결정자를 위한 리허설 랩으로, 정책 및 홍보 전략을 무위험으로 테스트할 수 있게 합니다.
- **미시적 관점**: 개인 사용자를 위한 크리에이티브 샌드박스입니다. 소설의 결말을 추론하든 상상력을 펼치는 시나리오를 탐색하든, 모든 것을 재미있고 즐겁게 접근할 수 있습니다.

진지한 예측부터 재미있는 시뮬레이션까지, 우리는 모든 "만약에"가 그 결과를 볼 수 있게 하여 모든 것을 예측 가능하게 만듭니다.

## 🚀 빠른 시작

### 방법 1: 소스 코드 배포 (권장)

#### 사전 요구 사항

| 도구 | 버전 요구 사항 | 설명 | 설치 확인 |
|------|---------|------|---------|
| **Node.js** | 18+ | 프론트엔드 런타임 (npm 포함) | `node -v` |
| **Python** | ≥3.11, ≤3.12 | 백엔드 런타임 | `python --version` |
| **uv** | 최신 버전 | Python 패키지 관리자 | `uv --version` |

#### 1. 환경 변수 구성

```bash
# 예제 구성 파일 복사
cp .env.example .env

# .env 파일을 편집하고 필요한 API 키를 입력하세요.
```

**필수 환경 변수:**

```env
# LLM API 구성 (OpenAI SDK 형식을 지원하는 모든 LLM API)
# 알리 클라우드의 qwen-plus 모델 권장: https://bailian.console.aliyun.com/
# 소모량이 많으므로 처음엔 40회 미만의 시뮬레이션으로 테스트 권장
LLM_API_KEY=your_api_key
LLM_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
LLM_MODEL_NAME=qwen-plus

# Zep Cloud 구성
# 매달 무료 한도로 간단한 사용 가능: https://app.getzep.com/
ZEP_API_KEY=your_zep_api_key
```

#### 2. 종속성 설치

```bash
# 모든 종속성 원클릭 설치 (루트 + 프론트엔드 + 백엔드)
npm run setup:all
```

또는 단계별 설치:

```bash
# Node 종속성 설치 (루트 + 프론트엔드)
npm run setup

# Python 종속성 설치 (백엔드, 자동으로 가상 환경 생성)
npm run setup:backend
```

#### 3. 서비스 시작

```bash
# 프론트엔드 및 백엔드 동시 시작 (프로젝트 루트에서 실행)
npm run dev
```

**서비스 주소:**
- 프론트엔드: `http://localhost:3000`
- 백엔드 API: `http://localhost:5001`

**개별 시작:**

```bash
npm run backend   # 백엔드만 시작
npm run frontend  # 프론트엔드만 시작
```

### 방법 2: Docker 배포

```bash
# 1. 환경 변수 구성 (소스 배포와 동일)
cp .env.example .env

# 2. 이미지 Pull 및 시작
docker compose up -d
```

기본적으로 루트 폴더의 `.env`를 읽고 포트 `3000(프론트엔드) / 5001(백엔드)`를 매핑합니다.

> 더 빠른 Pull을 위한 미러 주소는 `docker-compose.yml` 리포지토리에 주석으로 제공되므로 필요에 따라 교체할 수 있습니다.
